## Public Content

I focus on the methodological rigor, empirical validation, and the translation of theoretical optimization dynamics into practical, reproducible tools for pre-training data detection. 

## Overview
The paper introduces "Gradient Deviation Scores" (GDS) as a novel mechanism for pre-training data detection in Large Language Models (LLMs). Diverging from traditional approaches that rely on static likelihoods, perplexity, or fine-tuning loss heuristics, the authors reframe membership inference as an optimization-state identification problem. The central hypothesis is that a model reacts structurally differently when computing updates on familiar (pre-training) data versus unfamiliar data. 

To measure this without the massive computational overhead of multi-epoch fine-tuning, the authors inject a zero-initialized LoRA adapter into the target model. By executing a single forward and backward pass, they extract gradients exclusively with respect to the LoRA B matrices. From these gradients, an 8-dimensional feature vector is constructed, capturing the magnitude, eccentricity (spatial stabilization), and concentration (sparsity) of the parameter updates. These structural features are subsequently classified by a lightweight, supervised Multi-Layer Perceptron (MLP) to distinguish between members and non-members.

## Claimed Contributions
- A paradigm shift in membership inference from static likelihood analysis to dynamic, single-step optimization-state profiling.
- An efficient, theoretically grounded method for extracting gradient footprints using a single backward pass over zero-initialized LoRA B matrices, avoiding the need to modify base model weights.
- The formalization of an 8-dimensional gradient feature space derived from loss convergence theory, specifically utilizing update magnitude, location stabilization, and activation sparsity.
- Empirical superiority and strong cross-dataset transferability over existing state-of-the-art zero-shot baseline methods.

## Novelty And Prior Work
The transition from static log-probability metrics (such as Perplexity, Min-K%, or neighborhood-based likelihoods) to dynamic gradient analysis is a highly original and insightful step forward. Traditional likelihood-based methods are frequently confounded by sequence length variations, inherent word-frequency biases, and highly standardized text formats. Conversely, previous optimization-based methods often demand active fine-tuning on similarly distributed reference datasets, which is prohibitively slow and computationally expensive. 

The authors’ engineering choice to isolate the gradients of the zero-initialized LoRA B matrix is an elegant bridge between theory and practice. Because standard LoRA initialization natively yields non-zero gradients for the B matrix and zero for the A matrix in the first backward step, this technique effectively captures layer-wise error signals and neuron-level spatial patterns in a low-dimensional format. This drastically reduces the memory burden of storing full-parameter gradient matrices while preserving the structural footprint of the sample’s familiarity to the model.

## Technical Soundness
The theoretical foundation connecting model familiarity to parameter update dynamics is robust and well-argued. The paper successfully translates established optimization theories—such as Hessian spectrum reshaping and sparse activation convergence—into concrete empirical features. The analysis detailing the decay of update magnitude, the stabilization of update locations, and the increasing sparsity of parameter updates provides a mathematically sound justification for the chosen 8-dimensional feature vector.

However, the technical soundness is currently hindered by an unaddressed source of stochastic variance: the random initialization of the LoRA A matrix. Because the gradient with respect to the B matrix is mathematically dependent on A (e.g., $\nabla_B L = A^T x^T \nabla_h L$), the extracted gradient deviations are fundamentally stochastic. The manuscript does not currently formalize how this random seed influences the extracted features' variance or the downstream classifier's stability. 

Furthermore, the requirement to perform a backward pass introduces inherent systems-level challenges. Unlike forward-only likelihood methods, backpropagation necessitates caching all intermediate forward activations. In a batched setting, extracting independent gradient matrices for each sample is notoriously difficult and memory-intensive in standard PyTorch, as the default `.backward()` call aggregates gradients across the batch. Unless specialized vectorization tools (e.g., `torch.func` or Opacus) are explicitly utilized, this requirement risks heavily constraining throughput and potentially forcing a batch size of 1, which poses significant scaling challenges for massive LLMs (e.g., 70B+ parameters).

## Experimental Rigor
The empirical validation of the proposed framework requires deeper scrutiny, particularly concerning the fairness of baseline comparisons and the statistical stability of the dynamic measurements.

First, GDS fundamentally relies on a supervised (or semi-supervised) MLP classifier trained on a curated set of labeled member and non-member data. By contrast, standard baselines like Min-K% or Perplexity are typically deployed in a purely zero-shot, unsupervised manner. Comparing a supervised classifier—which inherently optimizes a decision boundary based on explicit labels—directly against zero-shot heuristics creates a structurally unfair evaluation. To strictly isolate the algorithmic advantage of the gradient features, the zero-shot baselines must be granted equivalent access to the calibration dataset to optimally tune their decision thresholds, or an MLP classifier must similarly be trained on baseline features (e.g., token-level log-probabilities).

Second, relying on an auxiliary MLP introduces the risk of domain overfitting and data leakage. If the MLP is trained on gradient features derived from a specific text domain, it may overfit to the idiosyncrasies of that domain's gradient footprint, artificially inflating cross-dataset transferability if strict domain separation between the training and testing splits is not enforced. 

Third, the dynamic curves presented to justify the temporal evolution of parameter updates over optimization epochs lack error bars, standard deviations, or statistical significance indicators. Without these variance bands, it is difficult to determine whether the observed stabilization in magnitude, eccentricity, and concentration is statistically reliable across different batches, sequence lengths, and random seeds.

## Decision-Relevant Strengths
- **Theoretical Elegance:** Grounding membership inference in Hessian spectrum reshaping, magnitude decay, and activation sparsity introduces a theoretically mature dimension to pre-training data detection, moving beyond simple loss heuristics.
- **Efficient Feature Extraction:** Using the zero-initialized LoRA B matrix to capture error signals via a single forward-backward pass effectively avoids the massive computational penalty of multi-epoch full-model fine-tuning.
- **Robustness to Static Artifacts:** By analyzing structural gradient distributions (location and concentration) rather than relying exclusively on loss magnitude (which strongly correlates with sequence perplexity), the method is theoretically insulated against the word-frequency biases that commonly confound logit-based detectors.

## Decision-Relevant Concerns
- **Strict White-Box Limitation:** The fundamental requirement for a backward pass limits this method strictly to open-weights architectures. It cannot be applied to proprietary API-constrained LLMs (even those exposing log-probabilities), which restricts its real-world applicability for widespread copyright and contamination auditing.
- **Unfair Baseline Comparisons:** Evaluating an explicitly trained MLP classifier against uncalibrated zero-shot baselines confounds the utility of the gradient features with the advantages of supervised learning.
- **Stochastic Variance:** The extracted gradient features are mathematically dependent on the random initialization of the LoRA A matrix, but the variance of the resulting GDS scores across multiple random seeds remains unquantified.
- **Hardware and Throughput Overhead:** The necessity of caching intermediate activations and computing per-sample gradients strictly increases the memory floor and compute latency compared to forward-only detectors, creating practical deployment bottlenecks.

## Missing Evidence Or Experiments
To rigorously validate the central claims, the following experimental evidence is required:
1. **Fair Baseline Calibration:** Empirical results showing the performance of baseline methods (like Min-K% or PPL) when they are calibrated using the identical member/non-member data split used to train the GDS MLP. 
2. **Variance Ablation:** An analysis of the MLP's prediction variance and the stability of the 8-dimensional gradient features across multiple random initialization seeds for the LoRA A matrix.
3. **Computational Overhead Benchmarks:** A quantitative table detailing the peak memory usage and throughput (e.g., samples processed per second) of GDS versus forward-pass baselines on standard hardware, alongside a clarification of how per-sample batched gradients are handled in the codebase.
4. **Cross-Domain Verification:** Explicit confirmation that the domains used to train the MLP have zero overlap with the evaluation sets, ideally demonstrating out-of-distribution transfer (e.g., training the MLP on structured code gradients and evaluating on unstructured prose).
5. **Statistical Confidence Metrics:** The inclusion of error bands on all dynamic optimization curves, and the reporting of standard deviations or statistical significance tests (e.g., paired t-tests) on the final AUC and TPR@5%FPR evaluation metrics.

## What Would Change My Assessment
My leaning is currently positive due to the strong conceptual contribution and the elegant execution of the LoRA feature extraction. However, this support is explicitly conditional on experimental fairness and statistical rigor. 

I would confidently advocate for acceptance if the authors can demonstrate that GDS maintains a robust performance margin over zero-shot baselines when those baselines are granted identical access to calibration data. Furthermore, providing a variance ablation addressing the stochasticity of the LoRA A matrix and including detailed computational overhead metrics would thoroughly solidify the technical soundness. 

Conversely, if the apparent performance gains are shown to be primarily attributable to the supervised nature of the MLP—or if the gradient features exhibit prohibitive variance across random initialization seeds—I would recommend rejection, as the core claims of structural reliability would be fundamentally unsubstantiated.
