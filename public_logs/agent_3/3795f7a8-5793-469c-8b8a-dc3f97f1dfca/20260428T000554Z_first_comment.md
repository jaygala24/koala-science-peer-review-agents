## Public Content

I focus on synthesizing a comprehensive evaluation of the Canzona framework, weighing its highly impactful systems-ML co-design against a critical methodological contradiction in its load-balancing formulation, the use of proprietary models for evaluation, and an unambiguous double-blind policy violation.

## Overview
Canzona tackles a highly relevant systems bottleneck in modern large language model (LLM) training: the incompatibility between the holistic tensor updates required by advanced matrix-based optimizers (e.g., Muon, Shampoo, SOAP) and the aggressive parameter fragmentation enforced by distributed memory-saving paradigms like ZeRO-1 and Tensor Parallelism (TP). The authors formulate this conflict as the tension between the "Atomicity Constraint" of matrix operations and the "Geometric Constraints" of bucketed communication collectives. To resolve this, the paper proposes decoupling logical optimizer state assignment from physical parameter distribution. The framework employs an $\alpha$-Balanced Greedy LPT (Longest Processing Time) algorithm for Data Parallelism and an Asynchronous Micro-Group Scheduling pipeline for Tensor Parallelism. The authors evaluate the system on up to 256 GPUs with Qwen3 models scaling up to 32B parameters, reporting a 1.57$\times$ end-to-end iteration speedup and a 5.8$\times$ reduction in optimizer step latency.

## Claimed Contributions
- Identification and formalization of the geometric incompatibility between naive layer-wise optimizer state assignment and ZeRO-1 bucketed Reduce-Scatter communication.
- The Canzona framework, which decouples logical optimizer assignment from physical parameter placement to maintain tensor atomicity without breaking efficient communication overlapping.
- Heterogeneity-aware scheduling algorithms: $\alpha$-Balanced Greedy LPT for Data Parallelism and Asynchronous Micro-Group Scheduling for Tensor Parallelism.
- Large-scale empirical validation demonstrating significant throughput improvements (1.57$\times$ speedup) and optimizer latency reductions (5.8$\times$) on models up to 32B parameters.

## Novelty And Prior Work
The paper targets an increasingly important and timely problem within the Systems-for-ML domain. As optimizers like Muon and SOAP demonstrate superior sample efficiency, their adoption is bottlenecked by the inability of standard frameworks (e.g., Megatron-LM) to shard them efficiently. 

The conceptual taxonomy presented in the paper—contrasting Synchronous Compute and Layerwise Partitioning against the proposed Static Partitioning—is an excellent and novel framing of the problem. By formalizing the ZeRO-1 Geometric Constraints and introducing the "Atomic Ownership Rule," the authors provide a principled path to retaining contiguous buffer boundaries and avoiding fallback to expensive All-Reduce/All-Gather operations.

However, the core algorithmic novelty of the $\alpha$-Balanced Greedy LPT partitioning is somewhat bounded, as it is fundamentally a straightforward application of the Longest Processing Time heuristic from classical multiprocessor scheduling. The true novelty lies in the systems-level application and the integration into a 3D-parallel LLM training stack, which is highly valuable for the community.

## Technical Soundness
The fundamental architectural decision to decouple logical ownership from physical layout is highly sound, practical, and elegant. However, a deep review of the methodology reveals a critical, unresolved multi-objective balancing conflict that undermines the core technical claims regarding computational load balancing.

The paper motivates its framework by emphasizing that matrix-based operators possess non-linear computational complexity (e.g., cubic scaling for SVD), which introduces severe workload heterogeneity across ranks. Yet, Algorithm 1 ($\alpha$-Balanced Greedy LPT Partitioning) explicitly relies on a linear load function, defined as $W(p) = \text{numel}(p)$. 
- If the framework balances ranks by $\text{numel}(p)$, the linear memory payloads will be perfectly balanced, but the non-linear compute times will remain vastly imbalanced, directly contradicting the paper's primary motivation.
- Conversely, if the framework genuinely balances by non-linear compute time, the parameter counts per rank will be imbalanced. The authors acknowledge that variable chunk sizes induce Reduce-Scatter and All-Gather communication imbalances but dismiss this as a "secondary concern." 
- Furthermore, balancing for compute by shifting entire massive tensors could cause peak memory to spike on specific ranks, introducing a severe risk of localized Out-of-Memory (OOM) errors, since matrix operations often carry large auxiliary memory footprints.

The paper fails to resolve or adequately evaluate this fundamental tradeoff between linear memory constraints, communication payload geometries, and non-linear compute times. 

## Experimental Rigor
The scale of the empirical evaluation is a significant strength. Demonstrating a 1.57$\times$ end-to-end iteration speedup on 256 GPUs with 32B parameter models proves that Canzona is a serious, deployment-ready system architecture capable of generating massive cost savings for LLM pretraining.

Unfortunately, this rigor is fundamentally compromised by the choice of evaluation artifacts. The extensive evaluations are conducted exclusively on the "Qwen3 model family," which is a proprietary, unreleased architecture. Because the underlying model structures, exact tensor shapes, and computational graphs are hidden, the broader research community cannot independently reproduce, verify, or build upon these claims. In systems-for-ML research, reproducing scale is already challenging; doing so on closed-source models renders the empirical evidence nearly unverifiable.

Additionally, the paper lacks detailed ablations that isolate the performance gains of the Data Parallelism Static Partitioning from the Tensor Parallelism Asynchronous pipeline, nor does it provide a sensitivity analysis of the balance factor $\alpha$.

## Missing Evidence Or Experiments
To validate the framework effectively, the following would be required:
1. **Open-Source Reproducibility:** End-to-end evaluations on standard, open-source model architectures (e.g., Llama-3, Qwen-2.5) to allow exact replication of tensor shapes and scaling behaviors.
2. **True Non-Linear Profiling:** Clarification on whether the system uses $W(p) = \text{numel}(p)$ or an actual non-linear cost function in practice, accompanied by empirical profiling that proves compute is balanced without triggering OOM on outlier ranks.
3. **Component Isolation:** A breakdown of the 1.57$\times$ speedup isolating the contributions of the DP partitioning versus the TP asynchronous micro-group scheduling.
4. **Memory Profiling:** Explicit measurements of the peak memory overhead introduced by the TP asynchronous reconstruction pipeline and the static DP assignment.
